# 会場運営チェックリスト — Chrome Built-in AI ミニラボ

**用途**: 講師・運営スタッフが前日〜当日の準備と実施をヌケモレなく行うためのリスト。
**印刷**: A4 両面印刷して当日持参推奨。

---

## 🕐 前日夜（〜24 時間前）

### 環境検証
- [ ] Chrome の**最新 Stable 版（148 以降）**を新規プロファイルで確認し、全 5 API が `available` になることを確認
- [ ] スターター配布 URL（GitHub Pages / Firebase Hosting）が公開されている
- [ ] Codelab サイト URL が公開されている
- [ ] スターター全 5 章が solutions を組み込んだ状態で動作する
- [ ] サンプル画像で名刺 OCR が期待通り動く
- [ ] 前日 Chrome 更新がある場合、更新後も API が動くか確認

### 資料準備
- [ ] メンター用チートシート × スタッフ人数 + 予備 2 部（A4 印刷）
- [ ] スターター URL / Codelab URL の QR コード ポスター（A3 印刷）
- [ ] 事前アナウンス済みリスト（参加希望者の連絡先）を運営に共有

### 予備機材
- [ ] 予備ノート PC 1〜2 台（環境設定済み・充電満タン）
- [ ] 電源タップ（4口以上を 2〜3 個）
- [ ] 電源ケーブル各種
- [ ] HDMI ケーブル（プロジェクタ接続用）

### 連絡経路
- [ ] 運営 Slack / Discord のスタッフチャンネルに全メンター join 済み
- [ ] 講師の当日連絡先を運営に共有

## 🆘 緊急時対応フロー

### ケース1: 会場 Wi-Fi が不安定 / モデル DL 中に切れる
1. 予備 PC の代替配布は諦め、代わりに **solutions を直接見せて概念だけ理解してもらう**モードに切替
2. スターター URL がロードできない場合、事前にダウンロードした ZIP をローカル配布

### ケース2: Chrome の新バージョンリリースで API 仕様変更が発覚
1. 前日夜または当日朝に発覚した場合、solutions/main.js を緊急修正
2. session 中に発覚した場合、Codelab サイトの該当箇所に注記追加
3. 参加者には「今日は概念中心に、詳細は後日修正版で」と説明

### ケース3: 参加者のスペックが大幅に不足
1. 予備 PC を貸与
2. 予備 PC が空きなしなら、隣席者とペアで見学する形に

### ケース4: 講師が緊急欠席
1. 副講師（メンターリーダー）が代講
2. Codelab サイトを画面共有して参加者に自習してもらう時間として活用
