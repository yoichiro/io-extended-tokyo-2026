# 🆘 緊急時対応フロー

## ケース1: 会場 Wi-Fi が不安定 / モデル DL 中に切れる
1. **solutions を運営 PC で映して概念だけ理解してもらう**モードに切替
2. スターター URL がロードできない場合、事前にダウンロードした ZIP をローカル配布

## ケース2: Chrome の新バージョンリリースで API 仕様変更が発覚
1. 前日夜または当日朝に発覚した場合、solutions/main.js を緊急修正
2. session 中に発覚した場合、Codelab サイトの該当箇所に注記追加
3. 参加者には「今日は概念中心に、詳細は後日修正版で」と説明

## ケース3: 参加者のスペックが大幅に不足
1. 隣席の参加者とペアで見学する形に案内
2. 「持ち帰り課題」として Codelab URL とスターター ZIP のリンクを共有し、後日環境を整えて挑戦してもらう

## ケース4: 講師が緊急欠席
1. 副講師（メンターリーダー）が代講
2. Codelab サイトを画面共有して参加者に自習してもらう時間として活用

# 📋 KPI 計測ポイント（振り返り用）

- 参加者数（登録 vs 実来場）
- 完了章数の分布（何章まで進めたか、アンケート回収）
- 「持ち帰って自分のアイデアで拡張したい」回答率（目標 70% 以上）
- 質問頻度が多かった章 / STEP のランキング
- SNS 投稿数
